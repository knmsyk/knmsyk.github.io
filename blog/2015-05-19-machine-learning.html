Supervised learning

What is supervised learning

  • To get a hypothesis, estimator or classifier of test samples (Task) from training samples (Experience).

Minimizing cost function

  • Choose coefficients θ so that hypothesis \(h (x)\) is close to output \(y\) of experience.
  • objective function, cost function (minimization), utility function (maximization) : \(J\)
    • sum of squared root error $$ J ( \theta ) = \sum_{i=1}^m ( \hxi ) - \yi )^2 $$

Gradient descent (GD)

repeat until convergence { $$ \theta_j : = \theta_j - \alpha \pd{ \theta_j }{ J ( \theta ) } $$ }

  • \(j = 0\)

$$ \pd{ \theta_0 }{ J ( \theta ) } = \frac 1 m \sumim ( \hxi - \yi ) $$

  • \(j \neq 0\)

$$ \pd{ \theta_j }{ J ( \theta ) } = \frac 1 m \sumim ( \hxi - \yi ) \xi_j $$

  • simultaneous update
  • can converge to a local minimum even with the learning rate α fixed.
  • \(J\) should decrease after every iterations.

Feature scaling

  • Make sure features are on a similar scale because non-scale features take long time to converge in GD.
  • learning rate should be same scale on any features.

Linear regression

  • Multivariable model
    • \(\hx = \theta_0 + \theta_1 x_1 + ... + \theta_n x_n = \ve \theta^\T \ve x\)

Normal equation (NE)

  • Method to solve for θ analytically
  • Set: \(J ( \theta ) = 0\)
  • Design matrix: \(X = \left[ \ve x^{(1)^\T}; \ve x^{(2)^\T}; ... ; \ve x^{(m)^\T} \right] \in \R^{m \times n}\)
  • \(\theta = ( X^\T X )^{-1} X^\T y\)

comparison GD and NE

  • GD
    • Need to choose α
    • Needs many iteration
    • Work well even when #features is large
  • NE
    • No need to choose α
    • Don’t need iteration
    • Need to compute \(( X^\T X )^{-1}\) : \(O ( n^3) )\)
    • Slow if #features is very large

non-invertivility

  • Reason. 1
    • Redundant features ( linearly dependent )
      • e.g. \(x_1 =\) size in \(feet^2, x_2 =\) size in \(m^2\)
  • Reason.2
    • Too many features
      • e.g. examples << features
      • –> Delete some features, or use regularization

Classification

Logistic Regression (LR)

  • hypothesis representation

    $$ \hx = g( \theta^\T x ) = \dfrac{ 1 }{ 1 + e^{- \theta^\T x } } $$

    • \(\hx \in [0, 1]\)
    • \(\hx\) : estimated probability that \(y = 1\) on input \(x\)
      • \(\hx = P(y = 1 | x; \theta)\)
      • e.g. \(h(\theta) = 0.7\) tell patient that 70% chance of tumor being malignant
  • Decision boundary
  • Cost function
    • \(\hx\) is modeled by \(P(y = 1 | x; \theta)\). In Linear regression, \(y\) is Real number, on the otherhand, in Logistic regression, \(y\) is \(0\) or \(1\).
      • We should consider a cost function that is suitable for the model.
    • \(y = 1\)
    • if \(y = 1\)

    $$ \cost( \hx, y ) = - \log ( \hx ) $$

    • if \(y = 0\)

    $$ \cost( \hx, y ) = - \log ( 1 - \hx ) $$

    • simplified

    $$ \cost( \hx, y ) = - y \log ( \hx ) - ( 1 - y ) \log ( 1 - \hx ) $$

    • minimization

    $$ \min \dfrac 1 m \sumim \cost( \hxi, \yi ) $$ $$ \min \dfrac 1 m \sumim \left(- \yi \log ( \hxi ) - ( 1 - \yi ) \log ( 1 - \hxi ) \right)$$

    • GD
      • Algorithm looks identical to linear regression!

    $$ \theta_j := \theta_j - \alpha \dfrac 1 m (\hxi - \yi) \xi_j $$

  • Multiclass classification
    • one-vs-all
      • \(\hcx\) : estimated probability that \(y = c\) on input \(x\)
        • \(\hcx = P(y = c | x; \theta)\)
        • On a new test input \(x\), pick the class \(c\) that $$\hat c = \argmax_c \hcx$$

Regularization

What for?

  • over-fitting / under-fitting: fail to generative to new examples
  • Addressing over fitting
    1. reduce number of features
      1. -> manually select features
      2. -> model selection algoritm
    2. Reguralization
      1. -> reduce magnitude/value of θ
      2. -> even if keep all a lot of features, works well
  • make the curve to smooth

Cost function

$$ J(\theta) = \dfrac{1}{2m} \sumim ( \hxi ) - \yi )^2 + \lambda \sumjn \theta_j^2 $$

  • λ is two large -> under-fitting
  • λ is two small -> over-fitting

GD

repeat until convergence { $$ \theta_j : = \theta_j - \alpha \pd{ \theta_j }{ J ( \theta ) } $$ }

  • \(j = 0\)

$$ \pd{ \theta_0 }{ J ( \theta ) } = \frac 1 m \sumim ( \hxi - \yi ) $$

  • \(j \neq 0\)

$$ \pd{ \theta_j }{ J ( \theta ) } = \frac 1 m \sumim ( \hxi - \yi ) \xi_j + \dfrac{\lambda}{m} \theta_j$$

  • -> \(\theta_j : = \theta_j ( 1 - \alpha \dfrac \lambda m ) - \alpha \frac 1 m \sumim ( \hxi - \yi ) \xi_j\)

NE

  • suppose \(m \leq n (X \in \R^{m \times n}: \sharp examples \times \sharp featues)\) and \(\lambda > 0\)
    • - $$\theta = \left( X^\T X + \lambda \fourfourdiag{0}{1}{:}{1} \right)^{-1} X^\T y$$

#- - $$\theta = \left( X^\T X + \lambda \fourfour{0}{}{}{} {}{1}{}{} {}{}{:}{} {}{}{}{1} \right)^{-1} X^\T y$$ #- - $$\theta = \left( X^\T X + \lambda \thrthr{0}{}{} {}{1}{} {}{}{1} \right)^{-1} X^\T y$$

Forward propagation

  • Start from input layer
  • Backpropagation
    • start form output layer
    • http://en.wikipedia.org/wiki/Backpropagation
      • For better understanding, the backpropagation learning algorithm can be divided into two phases: propagation and weight update.
        • Phase 1: Propagation
          • Each propagation involves the following steps:
            1. Forward propagation of a training pattern's input through the neural network in order to generate the propagation's output activations.
            2. Backward propagation of the propagation's output activations through the neural network using the training pattern's target in order to generate the deltas of all output and hidden neurons.
        • Phase 2: Weight update
          • For each weight-synapse follow the following steps:
            1. Multiply its output delta and input activation to get the gradient of the weight.
            2. Bring the weight in the opposite direction of the gradient by subtracting a ratio of it from the weight.

Precision and recall

predicted \ actual 1 0
1 true positive false positive
0 false negative true negative
  • Definition
    • Precision = true positives / no. of PREDICTED positives
      • Predicted accuracy
      • no. of PREDICTED positives = true positive + false positive
    • Recall = true positives / no. of ACTUAL positives
      • Actual accuracy
      • no. of ACTUAL positives = true positive + false negative
  • Examples
    • If we avoid misclassification
      • -> Higher precision, lower recall
    • If we avoid misdiagnosis
      • -> Lower precision, higher recall
